A-Level Statistics — Year 12 Crib Sheet

Edexcel · AS/Year 1 · Statistics
Data Collection Ch 1
Census vs sample
Census: every member of the population. Accurate but expensive, time-consuming, and may be impractical. Sample: a subset — cheaper, faster, but may not be representative.
Simple random
Every item has an equal chance of selection. Use random number generator/table from a sampling frame.
+ Bias-free, easy to implement
− Need a complete sampling frame; may not be representative of subgroups
Systematic
Take every kth item from an ordered list, where k = population size ÷ sample size. Random start point.
+ Simple, suitable for large populations
− Sampling frame needed; can introduce bias if there is a periodic pattern
Stratified
Population divided into strata (groups). Sample proportionally from each stratum.
Stratum sample = (stratum size / population size) × total sample size
+ Proportional representation of subgroups
− Strata must be clearly defined; still need a sampling frame
Quota
Interviewer selects a specified number from each subgroup until quota is met. Non-random — interviewer chooses who to approach.
+ No sampling frame needed; quick and cheap
− Bias from interviewer choice; not truly random; can't generalise
Opportunity (convenience)
Sample whoever is available at the time. Easiest method but least reliable.
+ Easy, cheap, fast
− Unlikely to be representative; very prone to bias
Exam questions love asking you to identify the sampling method used and justify advantages/disadvantages in context. Always relate your answer to the specific scenario given.
For stratified sampling calculations: always round your stratum samples to whole numbers and check they sum to the total sample size.
Data Presentation & Interpretation Ch 2–4
Histograms
Continuous data with unequal class widths. The y-axis is frequency density, not frequency.
Frequency density = frequency / class width
Area of each bar = frequency. Total area = total frequency.
Cumulative frequency
Plot upper class boundary vs cumulative frequency. Use to estimate median (n/2), lower quartile (n/4), upper quartile (3n/4). Draw smooth S-shaped (ogive) curve.
Box plots
Show minimum, Q1, median, Q3, maximum. Useful for comparing distributions side by side. Outliers plotted as individual crosses beyond the fences.
Mean
Ungrouped: x̄ = Σx / n
Grouped: x̄ = Σfx / Σf  (use class midpoints for x)
Median & mode
Median: middle value when ordered. For grouped data, use interpolation or read from cumulative frequency graph. Mode: most common value / modal class (highest frequency density for grouped data).
Variance & standard deviation
Variance: σ² = Σx²/n − (Σx/n)² = Σx²/n − x̄²
Grouped: σ² = Σfx²/Σf − (Σfx/Σf)²
Standard deviation: σ = √(variance)
Range & IQR
Range = max − min
IQR = Q3 − Q1
IQR is resistant to outliers; range is not.
Outliers
Outlier if: x < Q1 − 1.5 × IQR  or  x > Q3 + 1.5 × IQR
These are called fences. An outlier is not necessarily an error — it may be a genuine extreme value. State whether to include or exclude with justification.
Coding
If y = (x − a)/b then:
ȳ = (x̄ − a) / b
σ_y = σ_x / b
Coding simplifies calculations. Code, compute, then decode.
Comparing distributions
Compare a measure of central tendency (mean or median) and a measure of spread (IQR or standard deviation). Always interpret in context.
Example"The mean score for group A (54) is higher than group B (48), suggesting group A performed better on average. The IQR for A (12) is smaller than B (20), indicating less variation in group A's scores."
When comparing distributions, you MUST compare both an average and a spread. Comparing only one loses marks. Always use "on average" and "consistency/variation" language.
Probability Ch 5
Addition rule
P(A ∪ B) = P(A) + P(B) − P(A ∩ B)
For mutually exclusive events: P(A ∩ B) = 0, so P(A ∪ B) = P(A) + P(B).
Multiplication rule
P(A ∩ B) = P(A) × P(B|A)
For independent events: P(B|A) = P(B), so P(A ∩ B) = P(A) × P(B).
Conditional probability
P(A|B) = P(A ∩ B) / P(B)
"The probability of A given that B has already occurred." Restricts the sample space to B.
ExampleP(pass | revised) = P(pass ∩ revised) / P(revised)
Independent events
A and B independent ⟺ P(A ∩ B) = P(A) × P(B)
Equivalently: P(A|B) = P(A)
One event does not affect the probability of the other.
Mutually exclusive
A and B mutually exclusive ⟺ P(A ∩ B) = 0
Cannot happen at the same time. On a Venn diagram, the circles do not overlap.
Tree diagrams
Branches show sequential events. Multiply along branches for P(A ∩ B). Add between branches for P(A ∪ B). Probabilities on each set of branches must sum to 1.
Venn diagrams
Fill in from the intersection outward. Don't forget the region outside all sets (complement). Useful for "at least one" problems: P(at least one) = 1 − P(none).
Mutually exclusive events CANNOT be independent (unless one has probability 0). If P(A ∩ B) = 0 and both P(A), P(B) > 0, then P(A ∩ B) ≠ P(A)×P(B). Examiners test this distinction explicitly.
For Venn diagram problems: let the intersection be x and set up an equation. Often the total probability must equal 1, or a given number of items are "in neither".
Statistical Distributions Ch 6
Binomial conditions
X ~ B(n, p) requires:
1. Fixed number of trials (n)
2. Two outcomes only (success/failure)
3. Constant probability of success (p)
4. Trials are independent
Binomial formula
P(X = r) = ⁿCᵣ × pʳ × (1−p)ⁿ⁻ʳ
where ⁿCᵣ = n! / (r!(n−r)!)
Mean of binomial
E(X) = np
This is the expected number of successes. Variance = np(1−p).
Cumulative probabilities
P(X ≤ r) = P(X=0) + P(X=1) + ... + P(X=r)
Use cumulative binomial tables or calculator. For other forms:
P(X < r) = P(X ≤ r−1)
P(X > r) = 1 − P(X ≤ r)
P(X ≥ r) = 1 − P(X ≤ r−1)
Using tables
Tables give P(X ≤ r) for specific n and p values. If p > 0.5, use the complement: let Y = n − X, then Y ~ B(n, 1−p) where 1−p < 0.5.
The four binomial conditions are a favourite exam question. State them clearly, then verify each one holds in the context given. If any condition fails, justify why the binomial model may not be appropriate.
For P(X ≥ 1), always use 1 − P(X = 0). It's faster and less error-prone than summing individual terms.
Hypothesis Testing Ch 7
Null & alternative
H₀: p = k  (null — the default assumption)
H₁: p < k, p > k, or p ≠ k  (alternative)
H₀ always uses =. H₁ determines one-tailed (< or >) or two-tailed (≠).
Significance levels
Common levels: 1%, 5%, 10%. The significance level is the probability of incorrectly rejecting H₀ (Type I error). Lower level = stronger evidence required to reject.
One-tailed vs two-tailed
One-tailed: testing for increase or decrease (H₁: p < k or p > k). Entire significance level in one tail.
Two-tailed: testing for change in either direction (H₁: p ≠ k). Significance level split equally between both tails.
Critical region
The set of values of X that would lead to rejecting H₀. Found by comparing cumulative probabilities to the significance level.
For H₁: p < k → critical region in lower tail
Find largest r such that P(X ≤ r) < significance level
Critical value = r, critical region: X ≤ r
Binomial test steps
1. Define the parameter: "Let p = probability of..."
2. State H₀ and H₁ in terms of p
3. State significance level and distribution: X ~ B(n, p) under H₀
4. Calculate P(X ≤ observed) or P(X ≥ observed) under H₀
5. Compare p-value with significance level
6. Conclude: reject or do not reject H₀
7. Interpret in context
Actual significance level
The probability of being in the critical region under H₀. Due to discrete distributions, this is usually less than the stated significance level.
Actual significance = P(X is in critical region | H₀ true)
Interpreting results
If reject H₀"There is sufficient evidence at the _% significance level to reject H₀. There is evidence that [context interpretation of H₁]."
If do not reject H₀"There is insufficient evidence at the _% significance level to reject H₀. There is no evidence that [context interpretation of H₁]."
NEVER say "accept H₀" — always say "do not reject H₀" or "insufficient evidence to reject H₀". You cannot prove H₀ is true; you can only fail to disprove it.
Always define p clearly in context before stating hypotheses. Conclusion must be in context of the original question — don't just say "reject H₀", say what that means for the real-world scenario.
